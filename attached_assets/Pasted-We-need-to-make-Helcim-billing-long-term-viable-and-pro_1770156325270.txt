We need to make Helcim billing long-term viable and production-grade.

Non-negotiable product requirements

Clients choose a billing day (1–28) and are billed monthly on that day automatically.

The system supports one-time event charges (tournaments/camps/drop-ins).

The app must NOT run recurring charges. Helcim must perform the recurring billing automatically.

We must never store raw card/bank details. Helcim vault references only.

We must be resilient to missed webhooks, ACH settlement delays, retries, and out-of-order events.

Current state (confirmed)

We are currently Model B:

processDailyClubBilling() cron at 3 AM charges stored tokens

Manual /api/platform/billing/run

Webhook /api/webhooks/payments currently handles Helcim cardTransaction and uses Helcim signature spec correctly
We need to convert to true Model A.

A daily “reconciliation sync” job that does NOT charge anyone, it only:

D) Idempotency & state machine
